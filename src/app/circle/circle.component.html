[type]="'circle'"

<app-sh-spinner [type]="'circle'" [shape]="'starter'" [diameter]="'72'" [stroke]="'#ed1b1e'" [width]="'3'" [speed]="'1.2'"></app-sh-spinner>

Generates the following markup:

        <div _ngcontent-qst-3="" class="sh-spinner sh-spinner-circle">
          <svg _ngcontent-qst-3="" class="sh-spinner-circle fadeOut" width="900" height="96" viewBox="0 0 900 96">
            <g _ngcontent-qst-3="" transform="translate(450, 48)">
              <circle _ngcontent-qst-3="" cx="0" cy="0" fill="none" stroke-miterlimit="10" id="starter0" stroke="#ed1b1e" opacity="1" stroke-width="3" stroke-linecap="round" r="36" transform="rotate([movable target])" stroke-dasharray="[movable target]">
              </circle>
            </g>
          </svg>
        </div>

Almost all of the elements of the svg circle are available. Some of the naming seems odd, or wrong but that is because I wanted to avoid using standard attribute names. They may well require a rethink at some point.

The colours specified in the [stroke] attribute only accepts a hex colour. This is because, using either rgb, rgba, hls or hlsa throws a 'foo is not a function' error.

Adding opacity to the element is as simple as adding an [opacity] option in the markup.
<app-sh-spinner [type]="'circle'" [shape]="'starter'" [diameter]="'122'" [stroke]="'#ed1b1e'" [width]="'5'" [speed]="'2'" [opacity]="'.3'"></app-sh-spinner>

The circle type also lends itself to dashes of equal sizes. The additional element is [divs], short for 'divisions'. An odd number is the number of visible lines and an even number is the number of lines and gaps. The number should not be enclosed within "'n'", just "n" as it needs to be a number, which is odd as it's the only one that needs this but isn't the only number.

[divs]="2"
[divs]="3"
