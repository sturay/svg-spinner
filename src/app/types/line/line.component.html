Generates the following markup:

        <div _ngcontent-kyy-3="" class="sh-spinner sh-spinner-line">
          <svg _ngcontent-kyy-3="" class="sh-spinner-line fadeOut" width="900" height="3" viewBox="0 0 900 3">
            <g _ngcontent-kyy-3="" transform="translate( 0, 1.5)">
              <line _ngcontent-kyy-3="" fill="none" stroke-miterlimit="10" y1="0" y2="0" id="starter0" stroke="#bada55" opacity="1" stroke-width="3" stroke-linecap="round" x1="3" x2="897" stroke-dasharray="[moveable target]">
              </line>
            </g>
          </svg>
        </div>

The line is nice and simple, as the e is no need to use the [diameter] attribute as it sets its width to the parent element's width in the DOM. This is meant as an indeterminate progress bar and not a loading bar.

There are fewer options available for this type, but that is not an issue.

The colours specified in the [stroke] attribute only accepts a hex colour. This is because, using either rgb, rgba, hls or hlsa throws a 'foo is not a function' error.
